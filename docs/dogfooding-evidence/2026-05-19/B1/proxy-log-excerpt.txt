SECTION 1 — REAL CAPTURE (mechanism proof)

The Basecamp ModuleProxy logs every cross-module method call at INFO level
together with the verbatim `QVariantList` of arguments. The lines below were
written to the swap module's own log namespace (`[swap]`); the same shape
appears for `[delivery_module]`, `[capability_module]`, `[package_manager]`,
etc. — no per-method redaction is applied.

[2026-05-19 08:02:21.100] [info] [swap] ModuleProxy: callRemoteMethod "messagingInit" args: QList(QVariant(QString, "{\"listen_port\":0,\"portsShift\":2457}"))
[2026-05-19 08:02:23.636] [info] [swap] ModuleProxy: callRemoteMethod "loadEnv" args: QList(QVariant(QString, "<REDACTED_REPO_ROOT>/.env"))
[2026-05-19 08:02:23.638] [info] [swap] ModuleProxy: callRemoteMethod "fetchBalancesFromEnv" args: QList(QVariant(QString, "<REDACTED_REPO_ROOT>/.env"))

Note 1: `messagingInit` is a `Q_INVOKABLE std::string messagingInit(const std::string& configJson)`
declared in [`swap-module/src/swap_impl.h#L62`](//swap-module/src/swap_impl.h#L62).
The JSON is logged literally — every character of the configJson string
appears in the log line. Today this configJson happens to contain only
non-secret delivery settings, but the same proxy machinery applies to any
`std::string` argument on any cross-module method.

Note 2: `fetchBalancesFromEnv(path)` is the local mitigation shipped in
this repo (see B1 draft). The arg is a filesystem path, not a JSON blob.
Before this mitigation the equivalent line was
`callRemoteMethod "fetchBalances" args: QList(QVariant(QString, "<full configJson with secrets>"))`
(see Section 2 below).

Capability-token traffic at INFO level (same run, same log) — also useful
context for the redaction request:
